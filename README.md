# LoveAI

LoveAI 是一个以“浪漫表达”与“动态播放” 为核心的 Android 应用。它通过启动页、特效轮播、文案展示、音乐播放、方案编排、收藏动态和结尾页，组合出一套适合告白、纪念日、生日祝福、异地思念等场景的沉浸式表达流程。

当前仓库已完成三期建设，项目已经从“随机浪漫特效播放器”升级为“可编排、可保存、可复用的浪漫动态生成器”。

## 当前阶段

- 当前稳定分支：`codex/phase3-feature-expansion`
- 当前阶段定位：三期完成版

### 模块职责

- `ui`
  负责启动页、主播放页、方案编辑页、方案库、收藏页、结尾页和页面跳转。
- `viewmodel/LoveViewModel.kt`
  负责当前动态序列、播放索引、自动轮播状态、当前方案和当前收藏动态等核心状态。
- `repository/EffectRepository.kt`
  负责随机特效生成、按类型生成、按变体还原动态。
- `repository/PlanRepository.kt`
  负责方案保存、读取、删除与持久化。
- `repository/FavoriteRepository.kt`
  负责收藏整套动态，而不是单个页面。
- `manager/MusicManager.kt`
  负责音乐资源扫描、播放列表管理、顺序循环、切歌和绑定方案音乐。
- `ui/effects`
  负责 12 类浪漫特效的绘制、动画节奏、文字样式和场景氛围。

## 业务功能

### 1. 启动页

- 进入应用后展示启动动画与倒计时
- 自动初始化默认音乐播放列表
- 用于承接应用氛围和第一屏沉浸感

### 2. 主播放流程

- 使用 `ViewPager2` 播放一组动态页面
- 随机模式下每轮从 12 种特效中随机选 8 个
- 支持自动轮播、手动滑动、暂停继续
- 最后一页至少停留 10 秒后进入结尾页

### 3. 特效系统

当前支持 12 类基础浪漫特效：

- `HEART_RAIN`
- `FIREWORK`
- `STARRY_SKY`
- `PETAL_FALL`
- `BUBBLE_FLOAT`
- `TYPEWRITER`
- `HEART_PULSE`
- `RIPPLE`
- `SNOW_FALL`
- `METEOR_SHOWER`
- `BUTTERFLY`
- `AURORA`

每种特效包含多个 `EffectVariant`，差异不仅是文案，还包括：

- 主副色和背景色
- 动画速度与粒子密度
- 文案内容
- 局部氛围与节奏参数

### 4. 文字系统

- 主标题和副标题由 `BaseEffectView` 统一调度绘制
- 不同特效有不同的文字排版策略
- 已加入文字安全区，避免和底部按钮、音乐区重叠
- 三期仍保留后续继续强化的空间

### 5. 音乐系统

- 自动扫描 `res/raw` 音乐资源
- 支持播放、暂停、上一首、下一首、播放列表查看
- 默认使用顺序循环播放
- 支持方案绑定音乐并在播放时自动切换

### 6. 方案系统

- 支持自定义方案名称、主标题、副标题
- 支持从 12 种特效里手动选择，最少 5 个，最多 8 个
- 支持保存方案、再次打开方案
- 支持主题模板快速填充内容

当前已提供的模板方向包括：

- 告白
- 纪念日
- 生日
- 异地思念

### 7. 收藏系统

- 收藏的是当前整套动态，而不是单个特效页
- 收藏内容包括：动态序列、文案、绑定音乐、创建时间
- 收藏页支持打开整套动态和删除收藏
- 结尾页重新开始时，能够带回当前收藏动态继续播放

### 8. 结尾页

- 展示中文收尾文案
- 支持重新开始
- 支持系统分享
- 支持单独控制音乐播放状态

## 代码逻辑说明

### 随机播放逻辑

随机模式下采用“12 选 8”的策略：

1. 从 12 个特效类型中打乱顺序
2. 取前 8 个类型
3. 每个类型内部随机抽一个 `EffectVariant`
4. 生成一轮动态播放序列

这样可以保证：

- 每轮不会出现重复特效类型
- 每轮内容都有变化
- 不会一次塞入过多页面导致节奏拖长

### 方案播放逻辑

当用户从方案库打开一个方案时：

1. `MainActivity` 读取方案 ID
2. `LoveViewModel` 调用 `loadPlan`
3. `EffectRepository` 按方案里的特效类型生成对应页面
4. 如果方案绑定了音乐，则 `MusicManager` 切到指定歌曲

### 收藏动态逻辑

当用户点击收藏按钮时：

1. 读取当前整套动态的 `variantId` 列表
2. 读取当前文案和当前歌曲
3. 保存为一个 `FavoriteSequence`
4. 收藏页读取后可以完整还原这套动态

### 重新开始逻辑

- 普通随机播放：重新开始后重新随机一轮
- 方案播放：重新开始后继续播放当前方案
- 收藏动态播放：重新开始后继续播放当前收藏动态

## 四期规划

四期希望把项目从“可编辑的浪漫动态生成器”继续升级到“更完整、更强大的浪漫内容产品”。建议这一期重点做功能增强，而不是继续只做视觉微调。

### 1. 作品导出能力

- 导出当前动态为视频
- 导出为 GIF 或长图
- 生成分享封面和分享文案

### 2. 更完整的方案能力

- 方案封面图
- 方案标签与分类
- 方案排序、搜索、筛选
- 最近使用记录
- 方案复制和快速二次编辑

### 3. 更强的文案能力

- 每一页支持单独文案，而不是整套共用一组标题
- 提供多套高级字体与文字动画模板
- 支持“自动生成文案”或“按场景推荐文案”

### 4. 更强的音乐能力

- 导入本地音乐
- 每套方案支持专属播放列表
- 音乐淡入淡出与页面切换联动
- 音乐进度展示和拖动

### 5. 更完整的收藏与作品体系

- 收藏改为“作品集”
- 支持重命名、分组、批量删除
- 支持收藏作品再次编辑
- 支持按主题或时间归档

### 6. 模板市场化能力

- 扩展更多官方模板
- 节日专题模板
- 主题包 / 皮肤包
- 为后续在线素材更新预留结构

### 7. 用户引导与产品化体验

- 首次使用引导
- 空状态优化
- 操作反馈和错误提示统一
- 关键流程防误触和兜底恢复

## 推荐四期范围

如果四期希望“更强大一些”，但仍控制在一个合理阶段，建议优先做下面 5 项：

- 视频导出
- 方案封面与搜索筛选
- 每页单独文案
- 本地音乐导入
- 收藏作品再次编辑

这 5 项做完后，LoveAI 会从“可编排的动态表达工具”再进一步升级成“可生成、可编辑、可沉淀、可分享的完整作品型应用”。
